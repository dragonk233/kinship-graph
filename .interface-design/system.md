# 亲族图谱界面设计系统

## 方向与感受

这是一个供家庭成员整理人物、核对关系和理解亲属称谓的家谱工具。界面应当像一本正在使用的现代族谱册：安静、可信、有纸本温度，但仍保持工具软件所需的紧凑和明确。

领域语言来自族谱、世代、支系、称谓、血缘链、婚姻链与印章。避免通用 SaaS 仪表盘、悬浮指标卡和无意义装饰。

交互在移动端参考 Apple HIG 的触控、安全区和单任务呈现方式，在桌面端保留关系图工具需要的信息密度、精确指针操作和并列上下文。品牌视觉始终属于亲族图谱，不仿制原生系统外观。

## 视觉签名

“关系路径”与“当前主视角”共同构成产品签名：任何推导结果或关系写入都尽量显示为人物与基础事实组成的可读路径；人物、称谓和路径随主视角重新解释。朱砂用于标记当前视角、当前选择、关系路径方向和主要操作。

签名应出现在：

- 当前主视角印记；
- 画布人物与连接线；
- 称谓推导路径；
- 新增关系前的写入预览；
- 关系维护入口和选中状态。

## 色彩与材质

颜色来自宣纸米白、木版墨色、旧照片棕、朱砂印和族谱线灰。颜色必须承载结构、状态或关系含义，不使用装饰性渐变或多套强调色。

核心语义令牌位于 `src/design-system.css`：

```css
--surface-canvas: #eee9de;
--surface-paper: #f5f1e8;
--surface-raised: #faf7f0;
--surface-inset: #ede8dc;
--text-primary: #28251f;
--text-secondary: #5e584e;
--text-tertiary: #888074;
--border-default: rgba(59, 52, 42, .15);
--border-soft: rgba(59, 52, 42, .08);
--accent: #a53a2b;
--accent-strong: #812a20;
--wood: #765844;
--blood: #8d4a3c;
--marriage: #8e7a5c;
```

旧的 `paper`、`ink`、`line` 和 `cinnabar` 名称作为兼容别名保留，新增样式优先使用语义令牌。

## 层级与深度

采用轻边界和细微表面色差，避免厚重阴影。页面骨架由低透明度边界分隔；输入区域使用略深的 `surface-inset` 表现“承接内容”；浮层使用 `surface-raised`，只允许真正脱离页面层级的浮层使用克制的柔和阴影。

表面层级：

1. 关系画布：`surface-canvas`；
2. 页面与常驻面板：`surface-paper`；
3. 输入底：`surface-inset`；
4. 人物卡与浮层：`surface-raised`，浮层增加唯一的轻阴影层级。

## 字体

人物、称谓和标题使用宋体系，强化族谱与纸本文献感：`Songti SC`, `Noto Serif SC`, `STSong`, serif。

搜索、辅助说明、状态、元数据使用 Apple 系统字体或微软雅黑。路径和技术性关系序列使用等宽字体并启用等宽数字。层级不仅依赖字号，也要使用字重、字距和颜色区分。

## 间距、圆角与动效

基础间距单位为 4px。常用间距为 4、8、12、16、24、32px；禁止无理由出现不属于该尺度的随机间距。

圆角尺度：

- 小控件和标签：4–6px；
- 输入、按钮：6px；
- 人物卡：8px；
- 弹窗：10px；
- 胶囊形只用于数量与短状态。

动效快速、克制并使用减速曲线，不使用弹跳动画；必须尊重 `prefers-reduced-motion`。

## 核心组件模式

### 应用骨架

- 桌面使用人物索引、关系画布、称谓详情三栏工作台；
- `AppHeader` 承载品牌、主视角和全局工具，移动端折叠工具菜单；
- `MobileNavigation` 只承载图谱、人物、称谓三个一级任务；
- 人物行显示头像、姓名/生日和当前称谓，移动端点击后直接进入称谓页；
- 弹层桌面居中，移动端作为底部工作面板出现。

### 人物画布

- 人物卡显示姓名、动态称谓和必要的人物特征；
- 血缘使用实线，婚姻使用虚线；
- 当前视角用朱砂边框与“我”印记；
- 单击查看，双击切换视角；
- 快捷操作属于第二层，不持续占据画布；
- 画布支持拖动、缩放和适应全部人物。

### 关系录入

- 使用“关系分组 → 具体称谓 → 必要的连接人物”的顺序；
- 常用称谓必须转换成父母、子女或配偶等稳定基础事实；
- 未覆盖的关系通过“精确连接”录入，不不断扩张超长称谓列表；
- 提交前显示实际写入的关系链；
- 缺少必要中间人物时，给出原因并禁用提交；
- 移动端关系选项使用两列，桌面端最多三列。

### 关系维护

- 人物资料和人物关系分开维护；
- 关系维护只显示实际保存的基础连接，不把推导称谓伪装成可编辑事实；
- 移除关系不删除人物，并在操作后明确反馈；
- 删除人物必须使用单独的危险确认流程。

### 状态与反馈

- 交互控件必须具备默认、悬停、聚焦、选中、禁用状态；
- 朱砂用于主要操作、选择、警告和破坏性操作，但通过背景透明度区分强度；
- 保存、空数据、缺少连接人物和录入失败都必须有明确状态；
- Toast 简短描述结果，不承担需要长期阅读的信息；
- 状态不能只靠颜色表达，使用 `aria-current`、`aria-expanded`、`aria-pressed` 等语义说明状态。

## 跨端规则

### 桌面

- 三栏同时呈现，避免把关联上下文藏进多层页面；
- Hover 只作为精确指针设备的增强反馈，任何功能都必须可点击或通过键盘完成；
- 支持滚轮缩放、拖动画布、键盘焦点和可辨认的 focus-visible。

### 移动

- 820px 以下切换为图谱、人物、称谓三个一级任务，不压缩桌面三栏；
- 常用触控目标不小于 44px，输入字号至少 16px；
- 使用 `100dvh` 和安全区变量，底部导航避开 Home Indicator；
- 画布支持单指拖动和双指缩放，不依赖 Hover；
- 弹层从底部出现，操作区固定在面板底部；
- 520px 以下表单和关系选项改为适合触控的两列或单列；
- 不为适配而隐藏关键关系说明、错误提示或提交状态。

## 系统偏好与无障碍

- 支持减少动态效果、高对比、减少透明度和粗指针设备；
- 所有关键操作必须有可读名称和明确焦点；
- 触控与鼠标共享业务能力，但允许采用不同的操作编排；
- Hover 不得成为发现或完成核心功能的唯一方式。

## 设计禁区

- 不使用通用后台侧栏模板或 KPI 卡片；
- 不为每种支系或称谓分配装饰性色彩；
- 不直接保存“叔叔”“表姐”等可推导称谓作为事实关系；
- 不把人物编辑、关系编辑和删除人物混成一个无层级长表单；
- 不使用强边框、厚阴影、大圆角或弹跳动画；
- 不让关系结果只有结论而没有推导路径；
- 不把桌面界面等比例缩小后当作移动端。
